# Globe Assembly Notes

## 总体思路

Globe 被组织成球体、支架和底座三类部件。球体是单个完整球面；支架围绕球体形成不同形式的环或半环支撑；底座负责从支架下方提供圆形、方形、星形或桌式支撑。每个 class 先在自身局部坐标中完成装配，再统一应用整体位姿。

## 球体

球体没有子部件装配关系。它只表达一个完整的球面主体，半径直接决定球的尺寸，整体位置和旋转在 concept 级别处理。

## 支架

半环支架由竖向转轴、可选端点球和弧形支架组成。转轴可以是一整根，也可以拆成上下两段；端点球通过球极 anchor 贴到转轴端部。弧形支架保留自己的弧长和朝向定义，偏移只表示整条弧相对转轴中心的高度移动。

倾斜支架由横向转轴、半环和内部完整圆环叠加而成。它的重点是几个环件共享同一局部中心，但各自位于不同的支撑平面中。

封闭支架由一个或两个正交半环和一个完整圆环组成。半环数量决定支架是否只有一个方向的包围，完整圆环提供中间的封闭轮廓。

## 底座

圆柱底座和方形底座都是从局部顶部平面向下堆叠两层。上层的顶面位于局部原点，下层贴在上层底面下方。

星形底座由顶部圆盘和多条径向倾斜腿组成。腿围绕竖直轴等角分布，倾斜角同时决定单条腿的姿态和它相对圆盘底部的中心位置。

特殊底座由一根倾斜连接柱和一个完整圆环底座组成。连接柱的顶部固定在局部原点，圆环底座放在连接柱下端外侧，使圆环路径中心与倾斜连接关系保持一致。

桌式底座由顶部圆环、竖直腿和可选径向脚组成。腿围绕圆环等角下挂；如果有脚，每个脚与对应腿共享同一径向方向，并沿地面方向伸出。

## 验证说明

当前实现使用 `Sphere`、`Cylinder`、`Cuboid`、`Ring`、`Torus` 的 primitive 形状和 anchor/transform 能力表达装配。径向阵列通过 `radial_array_frames()` 表达；不再在 class 里手写每个阵列项的旋转矩阵。

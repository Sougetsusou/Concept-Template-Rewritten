# Door rewritten 装配说明

本文只记录 rewritten Door 的自然语言装配思路，不描述 legacy 公式迁移过程。

## 总体设计

Door 的局部几何由四类结构组成：

- 门板：单扇或双扇矩形板，按关闭姿态定义，再绕各自门轴开合。
- 门框：主三边框始终存在，前后副框可选。
- 合页：圆柱合页按竖直方向重复排列。
- 把手：不同形状的把手先在门面局部坐标中构建，再随对应门板绕门轴旋转。

重写时把“门轴”作为核心装配概念。门板和附着在门板上的把手都不再通过外部顶点变换表达开合，而是让 primitive 保持自身局部形状，并绕父坐标系中的门轴 pivot 旋转。

## 门板

门板的设计意图是：先确定关闭状态下门板中心和门轴位置，再让整块门板围绕门轴转动。单门和双门的区别只在关闭中心和门轴距离不同；开合本身仍是同一种绕 Y 轴 pivot 的关系。

实现备注：

- 使用 `Cuboid` 表达门板。
- 使用 `rotate_about_point()` 表达开合。
- 最终整体姿态保留 `YXZ` 和 offset-first 的门类姿态约定。

## 门框

门框的设计意图是：围绕门洞开口放置顶部横梁、左竖梁和右竖梁。主框提供基础门洞；前后副框是在深度方向附加的三边框。

装配思路：

- 顶部横梁的底面贴到开口上边。
- 左右竖梁的内侧底边贴到开口左右下角。
- 前后副框沿深度方向放到主框对应侧面附近。
- 内外偏移决定左右竖梁厚度和顶部横梁高度，这是门框自身的 shape rule。

实现备注：

- 使用三个 `Cuboid` 表达每个三边框。
- 使用 `bottom_center`、`bottom_left_center`、`bottom_right_center` 等 anchor 表达梁与开口边界的对齐。

## 合页

合页的设计意图是：从每组合页的底部基准点开始，沿竖直方向按“合页高度 + 间隔”重复放置圆柱。

实现备注：

- 使用 `Cylinder` 表达每个合页。
- 每个合页的 `bottom_center` 对齐到当前重复游标。
- 间隔序列只描述相邻合页之间的空隙。

## L 形把手

L 形把手的设计意图是：固定底座贴在门面上，竖向段向外伸出，横向段接在竖向段外侧形成 L 形端部。整组把手作为门板附属物随门开合。

实现备注：

- 使用三个 `Cuboid`。
- 每段先按门面法向贴合或堆叠。
- 完成局部形状后，所有段一起绕对应门轴旋转。

## Π 形把手

Π 形把手的设计意图是：两个连接脚贴近门面，上下分布；外侧主梁位于两个连接脚外侧并连接它们。把手整体随门开合。

实现备注：

- 使用三个 `Cuboid`。
- 连接脚贴门面，主梁贴到连接脚外侧。
- 该模板保留自己的前后面方向约定，因为它与其他把手类的符号约定不同。

## 圆柱把手

圆柱把手的设计意图是：多段圆柱沿门面法向从门面向外堆叠，形成圆柱形把手。

实现备注：

- 使用 `Cylinder`。
- 圆柱先旋转到主轴沿门面法向，再通过端面 anchor 串接。
- 串接完成后整组绕门轴开合。

## 球形把手

球形把手的设计意图是：圆柱底座和连接段从门面伸出，最外侧是球形或椭球形旋钮。

实现备注：

- 底座和连接段使用 `Cylinder` 端面 anchor 串接。
- 旋钮使用 `Sphere`。
- 旋钮的接触距离是该模板的局部形状规则，因此使用局部接触点贴到连接段末端。

## 边界

- 共享 geometry 层只承担 primitive、anchor、transform 和 assembly。
- Door 内部的门轴附着规则目前保留在 category 模板中；只有在其他 category 重复出现同一稳定语义后，才考虑提升为公共 relation。

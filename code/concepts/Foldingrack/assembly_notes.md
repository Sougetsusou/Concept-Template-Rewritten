# Foldingrack Assembly Notes

## 总体思路

Foldingrack 被组织成 rack 和 hook 两类部件。rack 表达可以绕同一侧铰点展开的两根长杆；hook 表达固定基座、伸出臂和末端弯钩的组合。每个 class 先在自身局部坐标中完成装配，再统一应用整体位姿。

## 折叠架

直折叠架由两根矩形杆组成。两根杆共享同一个铰点，一根从铰点向下展开，另一根从铰点向上展开；开合角只控制两根杆围绕深度方向的相反旋转。局部装配不再手算旋转后的杆中心，而是直接把杆上的边中心 anchor 放到公共铰点。

弯折叠架在直折叠架的基础上给两根杆的远端增加圆弧端部。每个圆弧的圆心位于对应杆远端外侧，外侧方向来自杆自身的表面法向；圆弧本身仍由 Ring 表达，Ring 的弧心 anchor 放到这个圆心。

## 挂钩

挂钩由基座、倾斜臂和半圆弯钩组成。基座用一侧面作为安装面；倾斜臂从基座另一侧伸出，并允许在高度和深度方向有安装偏移。半圆弯钩挂在倾斜臂末端下方，圆弧中心由臂末端表面点向下偏移得到。

## 验证说明

当前实现只使用 `Cuboid` 和 `Ring` 的 anchor/transform 能力表达装配。`relation-audit/categories/Foldingrack.md` 缺失，因此源码和参数样例是本 category 的主要 oracle。
